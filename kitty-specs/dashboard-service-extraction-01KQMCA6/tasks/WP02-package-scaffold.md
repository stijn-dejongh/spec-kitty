## Context

`src/specify_cli/dashboard/api_types.py` (440 lines) defines the TypedDict
response shapes for all dashboard routes. After extraction, the service layer
in `src/dashboard/` will import these types. If they remain in `specify_cli`,
the service layer would have to import from the CLI adapter package — creating
a circular dependency.

Moving `api_types.py` to `src/dashboard/api_types.py` and installing a shim
at the old path resolves this. The shim re-exports everything with a
`removal_release` annotation so future callers know when to update.

The `tests/architectural/test_layer_rules.py` maintains `_DEFINED_LAYERS` —
a frozenset of known top-level packages that the layer enforcement tests use.
`dashboard` must be added so the boundary test WP (WP05) can register its
assertion.
