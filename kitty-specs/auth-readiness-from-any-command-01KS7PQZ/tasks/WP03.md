## Implementation notes

- The coordinator's existing `try/except` swallowing in `evaluate_readiness` already wraps `_evaluate_uncached` ŌĆö if we let probe/renderer exceptions escape `_evaluate_uncached`, the result would be `_NOOP_DISABLED` for that invocation. We add narrower try/except blocks inside `_evaluate_uncached` so a probe failure degrades to `(UNKNOWN, None)` rather than dropping the entire ReadinessResult to disabled.
- The new coordinator code path also extracts `command_name = ctx.invoked_subcommand or "spec-kitty"` for the renderer (mirrors how the existing nag derives a command name).
- `probe_auth_status` import must be lazy at the call site to keep module-level imports cheap (mirroring Wave 1's `from specify_cli.saas.rollout import is_saas_sync_enabled` lazy pattern).
