# WP02 — Guidance renderer

## Scope

Render auth guidance per `OutputPolicy`. Reuse `_auth_recovery.emit_structured_stderr` for the non-interactive path.

## Files

- `src/specify_cli/readiness/render.py` — new module.
- `tests/readiness/test_auth_renderer.py` — new tests.

## Implementation outline

```python
def render_auth_guidance(
    *,
    status: AuthStatus,
    teamspace: str | None,
    command_name: str,
    output_policy: OutputPolicy,
) -> None:
    """Side-effect-only renderer for the auth probe's verdict.

    - INTERACTIVE + LOGGED_OUT_IN_TEAMSPACE → multiline rich panel on stderr.
    - NON_INTERACTIVE + LOGGED_OUT_IN_TEAMSPACE → emit_structured_stderr(...).
    - MACHINE_OUTPUT → no-op.
    - Any other status → no-op.
    - All exceptions swallowed.
    """
```

Interactive panel body:
- Title: "Logged out on a connected Teamspace"
- Body: `f"This repo is connected to Teamspace {teamspace}, but you are not logged in.\nRun: spec-kitty auth login"`
- Style: yellow border, expand=False.

## Test cases

1. `test_interactive_logged_out` — panel on stderr; contains `teamspace` handle and `spec-kitty auth login`. Stdout empty.
2. `test_non_interactive_logged_out` — single canonical line on stderr per `_auth_recovery.emit_structured_stderr` format. Stdout empty.
3. `test_machine_output_is_silent` — no stdout, no stderr.
4. `test_authenticated_status_is_silent` — passing `AUTHENTICATED` produces no output regardless of policy.
5. `test_not_in_teamspace_is_silent` — passing `NOT_IN_TEAMSPACE` produces no output.
6. `test_unknown_is_silent` — passing `UNKNOWN` produces no output.
7. `test_render_does_not_raise_on_panel_failure` — monkeypatch `rich.panel.Panel` to raise; ensure no exception escapes.
