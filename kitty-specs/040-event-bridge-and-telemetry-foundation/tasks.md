# Work Packages: EventBridge and Telemetry Foundation

**Inputs**: Design documents from `/kitty-specs/040-event-bridge-and-telemetry-foundation/`
**Prerequisites**: plan.md (required), spec.md (user stories), research.md, data-model.md
**Agent Profile**: Python Pedro — ATDD+TDD (Directives 016 & 017)

**Methodology**: Each WP follows ATDD+TDD: write acceptance/unit tests first (RED), implement to make them pass (GREEN), refactor. Tests and implementation live in the same WP — no separate "test-only" WP.

**Organization**: Fine-grained subtasks (`Txxx`) roll up into work packages (`WPxx`). Each work package is independently deliverable and testable.

**Prompt Files**: Each work package references a matching prompt file in `/tasks/` generated by `/spec-kitty.tasks`.

## Subtask Format: `[Txxx] [P?] Description`
- **[P]** indicates the subtask can proceed in parallel (different files/components).
- Paths are relative to repository root.

## Work Package WP01: Event Models & Bridge ABC (Priority: P0)

**Goal**: Create the foundational event type hierarchy (Pydantic frozen models) and the EventBridge abstraction layer (ABC, NullEventBridge, CompositeEventBridge). Tests written first per TDD.
**Independent Test**: `pytest tests/specify_cli/core/test_events/test_models.py tests/specify_cli/core/test_events/test_bridge.py` — all pass with ≥80% coverage.
**Prompt**: `/tasks/WP01-event-models-and-bridge-abc.md`
**Estimated prompt size**: ~500 lines

### Included Subtasks
- [ ] T001 **RED**: Write acceptance tests for event models in `tests/specify_cli/core/test_events/test_models.py` — model creation, JSON serialization round-trip, immutability (frozen), type discriminator, optional field defaults
- [ ] T002 **RED**: Write acceptance tests for bridge in `tests/specify_cli/core/test_events/test_bridge.py` — NullEventBridge discards silently, CompositeEventBridge fan-out, per-listener error isolation
- [ ] T003 **GREEN**: Create `src/specify_cli/core/events/__init__.py` with public API exports
- [ ] T004 **GREEN**: Create `src/specify_cli/core/events/models.py` — BaseEvent, LaneTransitionEvent, ValidationEvent, ExecutionEvent
- [ ] T005 **GREEN**: Create `src/specify_cli/core/events/bridge.py` — EventBridge ABC, NullEventBridge, CompositeEventBridge
- [ ] T006 **REFACTOR**: Run `ruff check` and `pytest` — ensure all tests green, clean code

### Implementation Notes
- **ATDD first**: T001 and T002 define the acceptance criteria as executable tests. They MUST fail initially (RED).
- **Then implement**: T003–T005 make the tests pass (GREEN).
- **Then clean up**: T006 refactors and validates.
- Use Pydantic BaseModel with `model_config = ConfigDict(frozen=True)` for all event models
- BaseEvent defines `timestamp: datetime` and `type: str` (Literal discriminator per subclass)
- CompositeEventBridge catches exceptions per-listener, logs via `logging.getLogger(__name__)`, continues dispatching

### Parallel Opportunities
- None — this is the foundation WP. All other WPs depend on it.

### Dependencies
- None (starting package).

### Risks & Mitigations
- Pydantic version mismatch: spec-kitty already depends on pydantic >=2.0 — verify in pyproject.toml
- Frozen model prevents mutation: Intentional per spec — events are immutable

## Work Package WP02: JSONL Writer & Factory (Priority: P0)

**Goal**: Create the JSONL event log writer and config-driven factory. Tests written first per TDD.
**Independent Test**: `pytest tests/specify_cli/telemetry/test_jsonl_writer.py tests/specify_cli/core/test_events/test_factory.py` — all pass with ≥80% coverage.
**Prompt**: `/tasks/WP02-jsonl-writer-and-factory.md`
**Estimated prompt size**: ~450 lines

### Included Subtasks
- [ ] T007 **RED**: Write acceptance tests for JSONL writer in `tests/specify_cli/telemetry/test_jsonl_writer.py` — write event to file, verify valid JSON per line, verify all fields, graceful failure on unwritable path
- [ ] T008 **RED**: Write acceptance tests for factory in `tests/specify_cli/core/test_events/test_factory.py` — NullEventBridge when no config, CompositeEventBridge when enabled, handles malformed config
- [ ] T009 **GREEN**: Create `src/specify_cli/telemetry/__init__.py` with public API exports
- [ ] T010 **GREEN**: Create `src/specify_cli/telemetry/jsonl_writer.py` — JsonlEventWriter with graceful failure handling
- [ ] T011 **GREEN**: Create `src/specify_cli/core/events/factory.py` — `load_event_bridge(repo_root)` config-driven factory, wired to JsonlEventWriter
- [ ] T012 **REFACTOR**: Run `ruff check` and `pytest` — ensure all tests green, clean code

### Implementation Notes
- **ATDD first**: T007 and T008 define acceptance criteria. They MUST fail initially (RED).
- **Then implement**: T009–T011 make the tests pass (GREEN).
- Config schema extension in `.kittify/config.yaml`:
  ```yaml
  telemetry:
    enabled: false
    log_path: ".kittify/events.jsonl"
  ```
- Factory returns NullEventBridge on any config error (missing file, malformed YAML, missing keys)
- JsonlEventWriter.handle() uses `event.model_dump_json()` + newline, wrapped in try/except OSError

### Parallel Opportunities
- T007 and T008 can be written in parallel (different test files).

### Dependencies
- Depends on WP01 (event models and bridge ABC).

### Risks & Mitigations
- File permission errors on JSONL write: Wrapped in try/except with warning log — spec FR-006
- Config.yaml doesn't exist: Factory gracefully returns NullEventBridge

## Work Package WP03: Orchestrator & CLI Wiring (Priority: P1)

**Goal**: Wire EventBridge into the orchestrator and CLI `move_task` so lane transitions emit events. Integration test written first per ATDD.
**Independent Test**: `pytest tests/specify_cli/core/test_events/test_integration.py` — lane transition with telemetry enabled produces correct JSONL event; disabled telemetry produces no events file.
**Prompt**: `/tasks/WP03-orchestrator-and-cli-wiring.md`
**Estimated prompt size**: ~400 lines

### Included Subtasks
- [ ] T013 **RED**: Write acceptance/integration test in `tests/specify_cli/core/test_events/test_integration.py` — full lane transition → JSONL event recorded; disabled telemetry → no file created
- [ ] T014 **GREEN**: Add `event_bridge` field to `ExecutionContext` dataclass in `src/specify_cli/orchestrator/executor.py`
- [ ] T015 **GREEN**: Wire `load_event_bridge()` call in orchestrator entry point (`src/specify_cli/orchestrator/integration.py`)
- [ ] T016 **GREEN**: Emit `LaneTransitionEvent` from `move_task` in `src/specify_cli/cli/commands/agent/tasks.py` after lane change succeeds
- [ ] T017 **REFACTOR**: Run full test suite (`pytest`), `ruff check` — verify backward compatibility, all existing tests still pass

### Implementation Notes
- **ATDD first**: T013 defines the end-to-end acceptance test. It MUST fail initially (RED).
- **Then wire**: T014–T016 implement the wiring to make integration test pass (GREEN).
- T014: `event_bridge: EventBridge = field(default_factory=NullEventBridge)` on ExecutionContext
- T016: After `set_scalar(wp.frontmatter, "lane", target_lane)` in move_task, load bridge and emit event
- All wiring defaults to NullEventBridge — zero impact on existing tests

### Parallel Opportunities
- T014 modifies executor.py, T016 modifies tasks.py — can be done in parallel after T013.

### Dependencies
- Depends on WP01 (bridge ABC) and WP02 (factory function).

### Risks & Mitigations
- Breaking existing tests: NullEventBridge is the default — unchanged code paths
- Import cycle: core.events has no orchestrator imports — safe
